# Claude Verifier

Claude 模型真伪鉴定工具 — 通过多维度测试判断 API 背后的模型是否为真正的 Claude（重点检测 Opus 4.6 / 4.7）。

## 背景

许多第三方 API 渠道声称提供 Claude Opus，但实际可能是其他模型或降智版本。本工具基于 [linux.do 社区 @Qiner 的研究](https://linux.do/t/topic/1765296)，利用 Claude 各版本已知的行为特征与分词器指纹进行真伪鉴定与版本区分。

## 测试项目

| # | 测试 | 原理 | 判定标准 |
|---|------|------|----------|
| 01 | 中文引号检测 | Claude 4.6 及更早版本分词器无法输出中文引号 `""`，Opus 4.7 已解除此限制 | 能输出 → 疑似 4.7；不能输出 → 疑似 4.6 及更早 |
| 02 | 乱码美咲检测 | Opus 在日文场景下有固定角色命名模式，旧版本伴随乱码 | Misaki + 无乱码 → 4.6/4.7；Misaki + 乱码 → 4.5 |
| 03 | 中文思考链检测 | 只有 Opus 能遵循「用中文思考」的指令 | 思考链中文占比 > 30% = Opus |
| 04 | 代码能力检测 | 通过生成礼花特效代码评估模型能力水平 | 12 项特征评分，≥75% 为 Opus 水准 |
| 05 | 模型身份指纹 | 探测模型自我认知、知识截止日期、创建者 | 辅助判断（可被 system prompt 伪造） |
| 06 | 水果逻辑推理 | 鸽巢原理题，考验深度推理与思考预算 | 答 21 = 满血 Opus，答 29 = 降智，答 34 = 严重降智 |
| 07 | Base64 编解码检测 | 中文 Base64 解码考验分词器完整性 | 完全正确/相似/乱码三档评分 |

## 使用方法

1. 直接用浏览器打开 `index.html`（纯前端，无需后端）
2. 配置 API Endpoint、API Key 和模型名称（默认 `claude-opus-4-6`）
3. 可点击 Model Name 右侧的 🔄 按钮**一键拉取模型列表**，自动填入最接近 `opus-4-7` / `opus-4-6` 的模型
4. 点击「测试连接」确认配置正确
5. 点击「运行全部测试」或单独运行某项测试
6. 可保存多套配置（Profile），方便切换不同渠道进行对比

### 附加功能

- **聊天面板**：支持自由对话与流式 Extended Thinking 查看，可用于手动验证模型表现
- **礼花代码运行**：Test 04 生成的代码可一键执行，直观对比模型产出质量

## API 格式

当前仅支持 **Anthropic 原生 API**（`/v1/messages`），启用了 `adaptive thinking` + `interleaved-thinking-2025-05-14` beta，max_tokens 默认 128k 以匹配 Opus 4.6/4.7 的最佳表现。

## 项目结构

```
├── index.html      # 主页面（UI + 测试定义）
├── css/style.css   # 样式（深色主题）
└── js/app.js       # 测试引擎 + API 客户端 + 流式解析
```

无构建流程、无依赖、纯 HTML/CSS/原生 JS。

## 判定结果

综合各项测试给出整体判定：

- **满血 Claude Opus** — 中文思考链 + 水果题答 21 + 中文引号符合版本预期
- **真 Opus 疑似降智** — 是 Opus 但水果题答错（思考预算被削减）
- **可能是 Claude 非 Opus** — 通过 Claude 特征但 Opus 专属项失败，可能是 Sonnet
- **高度可疑 / 无法确定** — 多项关键测试未通过

## 注意事项

- API Key 仅存储在浏览器 localStorage 中，不会发送到任何第三方服务
- Test 06 满血 Opus 需约 2 分钟思考时间，请耐心等待
- 单项测试无法作为最终判断依据，需综合多项结果
- 水果题与中文思考链在 Anthropic 原生 API 下效果最佳
