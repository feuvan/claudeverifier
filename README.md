# Claude Verifier

Claude 模型真伪鉴定工具 — 通过多维度测试判断 API 背后的模型是否为真正的 Claude（特别是 Opus）。

## 背景

许多第三方 API 渠道声称提供 Claude Opus，但实际可能是其他模型或降智版本。本工具基于 [linux.do 社区 @Qiner 的研究](https://linux.do/t/topic/1765296)，利用 Claude 模型的已知行为特征进行真伪鉴定。

## 测试项目

| # | 测试 | 原理 | 判定标准 |
|---|------|------|----------|
| 01 | 中文引号检测 | Claude 的安全分词器无法输出中文引号 `""` | 能输出 = 100% 非 Claude |
| 02 | 乱码美咲检测 | Opus 在日文场景下有固定角色命名模式和乱码特征 | Misaki 模式 + 乱码情况区分 Opus 4.5/4.6 |
| 03 | 中文思考链检测 | 只有 Opus 能遵循「用中文思考」的指令 | 思考链中文占比 > 30% = Opus |
| 04 | 代码能力检测 | 通过生成礼花特效代码评估模型能力水平 | 12 项特征评分，≥75% 为 Opus 水准 |
| 05 | 模型身份指纹 | 探测模型自我认知、知识截止日期、创建者 | 辅助判断（可被 system prompt 伪造） |
| 06 | 水果逻辑推理 | 鸽巢原理题，考验深度推理和思考预算 | 答 21 = 满血 Opus，答 29 = 降智，答 34 = 严重降智 |

## 使用方法

1. 直接用浏览器打开 `index.html`
2. 配置 API Endpoint、API Key 和模型名称
3. 选择 API 格式（Anthropic 原生 / OpenAI 兼容）
4. 点击「测试连接」确认配置正确
5. 点击「运行全部测试」或单独运行某项测试

支持的 API 格式：
- **Anthropic 原生 API** — 完整支持所有测试，包括 Extended Thinking
- **OpenAI 兼容格式** — 支持基础测试，思考链相关测试可能受限

## 项目结构

```
├── index.html      # 主页面
├── css/style.css   # 样式

```

## 注意事项

- API Key 仅存储在浏览器 localStorage 中，不会发送到任何第三方服务
- 测试 03（中文思考链）和测试 06（逻辑推理）在使用 Anthropic 原生 API 时效果最佳
- 测试 06 满血 Opus 需约 2 分钟思考时间
- 单项测试无法作为最终判断依据，需综合多项结果
